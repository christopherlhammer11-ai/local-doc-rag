# Local Doc RAG

**Private document Q&A for local PDFs and notes.** Local Doc RAG is a local-first document research app for asking questions across private files with source-backed answers.

Demo: **Watch the demo:** [Local Doc RAG](https://christopherhammer.dev/assets/videos/narrated/project-demos/local-doc-rag-narrated.mp4)

## Who Uses It

- Researchers with large PDF folders
- Compliance teams searching internal policy docs
- Students studying from notes and textbooks
- Consultants reviewing client files
- Operators who need answers from private documents without uploading them to a generic chatbot

## Core Features

- Local document ingestion path
- Private question-answering workflow
- Source-grounded responses
- Next.js app surface
- Local-first product positioning
- Built as a proof app for practical RAG workflows

## Why It Matters

Most "chat with your docs" tools assume the user is fine uploading sensitive files. This project is positioned around the opposite constraint: keep the document workflow local and make answers traceable.

## Quick Start

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).
